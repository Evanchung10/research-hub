# Research Hub Project Context

## 啟動句

使用者說出以下句子時，代表要啟動研究總站與文獻匯入工作流程。

主要啟動句：

```text
請啟動研究文獻模式
```

模糊但應視為同一意圖的說法：

```text
我要匯入一篇新的研究文獻
幫我把這篇文獻放進研究資料庫
這篇也要整理成之前那種文獻網頁
幫我用研究總站的格式處理這篇
照之前 C1 那個文獻流程做一篇新的
我要新增一篇文獻到 research hub
幫我做新的文獻導讀頁，並放進總站
這篇文獻要加入我的研究總站
用我的研究文獻資料庫流程處理
跟上次 design ethics 那篇一樣整理
```

## 防呆確認規則

偵測到主要啟動句或上述模糊句型時，不要立刻讀取文件或開始執行。應先向使用者確認一次，例如：

```text
我偵測到你可能要啟動研究文獻模式。是否確認啟動？確認後我會讀取 PROJECT_CONTEXT.md 與 LITERATURE_IMPORT_GUIDE.md，再開始處理。
```

只有在使用者明確回覆「同意」「確認」「是」「可以」「開始」等肯定語句後，才讀取本檔與 `LITERATURE_IMPORT_GUIDE.md`，並開始處理新的文獻、研究主題或網站更新。

## 專案定位

- 研究總站：`https://research.talentostudio.com`
- 單篇文獻範例：`https://design-ethics.talentostudio.com`
- 研究總站 repo：`Evanchung10/research-hub`
- C1 單篇文獻 repo：`Evanchung10/design-ethics-c1`

研究總站負責索引、分類、研究題目與文獻入口。單篇文獻頁負責原文、原文譯文、白話意譯、旁註、個人筆記、標註與匯出匯入。

## 研究主題

目前研究總站包含三個方向：

- 《墨子》兼愛與節用思想中的公共利益原則及其對設計倫理的啟示
- 地方創生政策變遷與影響
- 品牌視覺語法於 Lovart 之驗證

## 寫作與旁註語氣

- 使用正式、專業、清楚、可理解的繁體中文。
- 旁註內容應像研究者自己的導讀筆記，不使用「可幫你」「建議你」這類 AI 對使用者說話的語氣。
- 可使用「可用來」「可作為」「此段指出」「此處可連接」等主觀導讀語氣。
- 避免過度使用「不是……而是……」「與其……不如……」等重複對比句型。

## 翻譯規則

單篇文獻頁應同時保留：

- `original`：英文原文。
- `translation`：原文譯文，較貼近英文原文結構，用於查證。
- `plainTranslation`：白話意譯，依中文閱讀習慣重寫，用於閱讀與報告。

白話意譯規則：

- 把英文長句拆成 2 到 4 個中文短句。
- 英文被動句改成中文主動句。
- 抽象名詞不要硬翻，改成動作或關係。
- 遇到 `it is... that...`、`the process by which...` 等英文結構，直接改寫成中文論述。
- 保留關鍵術語，例如 `social ethics`、`design process`、`stakeholders`，第一次出現時用中文解釋。
- 避免太多「其」「此」「該」「乃」「所謂」等翻譯腔。
- 可視需要在段尾補一句白話收束，但不要每段機械式重複。

## 個人資料邊界

單篇文獻頁的個人筆記、標註、個人標籤、閱讀狀態存在使用者瀏覽器的 `localStorage`。

- 推送 GitHub 或 Netlify 不會改動其他人的本機筆記。
- 不同瀏覽器、不同裝置之間不會自動同步。
- 重要筆記應使用網站內的「匯出資料」備份。
- 個人標籤輸入支援 hashtag，例如 `#兼愛 #節用 #公共利益`。
